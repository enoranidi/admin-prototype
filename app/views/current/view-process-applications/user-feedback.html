User feedback session (consolidated)

Exercise is live

  • All users were happy with the layout of the application page example
  • For Full information tab:
    • For Character information, 1 user wasn't sure how this would be displayed, if a candidate had selected 'Yes'
    • For Equality and diversity information, 1 user questioned how we would capture information in the ethnicity sections, if candidates selected the ‘Prefer not to say’ option as this is needed by the diversity team
    • For Qualifications, 1 user mentioned about adding in another section for new solicitors and barristers
    • For Post-qualification experience, 1 user mentioned that this section should be more descriptive e.g. full or part-time, gaps in employment, working 11 days a month
    • For Independent assessors, 1 user mentioned the need for a title e.g. if someone is senior they would be addressed as ‘Master John Smith’ and ‘Master in the role of XXXX’
  • For Panel pack tab:
    • For Independent assessors, 1 user mentioned this wasn’t required
    • For Curriculum vitae (CV), 1 user questioned if the candidates CV would need to be edited to strip out any information about them

Exercise has now ended

  • All users were able to complete the 3 tasks, using the checkboxes and Set status button
    • However 1 user clicked on the link and not the checkboxes when completing the individual task. They thought if it was only one candidate to select, then to use a link and if more than one then to use the checkboxes
    • The same user found the bulk feature useful and that you could ‘remove’ and see who you had added
  • 1 user didn't like the word 'Selected' as a stage name and suggested ‘Selection Day’. They also mentioned the following:
    • Candidates who had ‘Passed scenario test’ to move to the next stage (i.e. Shortlisted) but still be present on the ‘Applied’ stage
    • More options for statuses in the ‘Applied’ stage, and mentioned that ‘Submitted first test(s)’ was irrelevant as it's automatically marked
  • 1 user commented about moving the Set status button to sit under the Applied heading
  • 2 users mentioned adding help or guided instructions (on what to do on these pages)
  • 3 users expected to see candidates move from one stage to the next one and not to see them in both stages
    • However 1 of them asked what happens if you selected the wrong person? Can you revert back?
    • Another user asked if we could download a record of applied, shortlisted etc before they get moved? As they were concerned about any status update errors

Different states

  • For Applied (984 applications... pages:
    • All users were happy with the intended functionality for those two examples
    • 1 user would like to sort by first and last name
    • 1 user asked about doing an advanced search e.g. 'multiple names and status types' at the same time
